DraftKings Contest Finder Agent - Problem Statement

Overview

The DraftKings Contest Finder Agent is an autonomous system designed to identify and notify users of potentially advantageous contest opportunities on the DraftKings platform. By analyzing contest details and participant experience levels, the agent aims to find contests where users may have a competitive edge.

Key Challenges

  1. Data Acquisition: Regularly fetch and parse contest data from DraftKings' API and web pages.
  2. Contest Filtering: Identify contests meeting specific criteria (e.g., fewer than 10 entrants, "Double Up" contests).
  3. Participant Analysis: Evaluate the experience level of contest entrants.
  4. Real-time Processing: Continuously update and analyze contest data to provide timely notifications.
  5. Notification System: Deliver contest recommendations to users via Slack in a clear and actionable format.
  6. Data Management: Store and manage contest and participant data efficiently.
  7. Authentication: Securely authenticate with DraftKings to access necessary data.
  8. Scalability: Handle multiple sports and a large number of contests simultaneously.

Project Goals

  1. Develop a robust web scraping system to collect contest data from DraftKings.
  2. Implement an efficient algorithm to analyze contest entrants and their experience levels.
  3. Create a reliable database system to store and manage contest information.
  4. Build a notification system that sends timely alerts to users via Slack.
  5. Ensure the agent operates autonomously, updating data and sending notifications every 5 minutes.
  6. Develop a system that is scalable across multiple sports on the DraftKings platform.

Success Criteria

The project will be considered successful if it can:

  • Accurately identify contests meeting the specified criteria (fewer than 10 entrants or "Double Up" contests).
  • Correctly analyze the experience level of contest participants.
  • Reliably notify users of contests where less than 30% of entrants are at the highest experience level.
  • Operate continuously, updating data and sending notifications every 5 minutes.
  • Scale to handle multiple sports available on DraftKings.
  • Maintain data integrity and respect DraftKings' terms of service and API usage limits.

Constraints and Considerations

  • The agent must be developed using Python 3.11+ and utilize Conda for virtual environment management.
  • AgentQL will be used for web scraping tasks.
  • Supabase will serve as the database solution.
  • The project timeline is set for 3 weeks, with coding to begin within 3 days of project initiation.
  • The system must be designed to avoid overloading DraftKings' servers or violating their terms of service.
  • User privacy and data security must be maintained throughout the operation of the agent.

This problem statement serves as the foundation for the DraftKings Contest Finder Agent project. It outlines the key challenges, goals, and success criteria that will guide the development process. As the project progresses, this document may be updated to reflect any changes in scope or requirements.
